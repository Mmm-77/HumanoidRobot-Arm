# 基于 AprilTag 视觉定位的四自由度机械臂上位机项目规划书

## 1. 项目概述

本项目开发一套运行于 Linux、ROS 2 和 Python 环境的机械臂上位机。AprilTag 固定在墙面，操作者手持相机移动；上位机通过 AprilTag 估计相机在空间中的位姿，并将相机相对初始位姿的变化映射为机械臂末端运动，使末端跟随相机移动。

系统采用相对跟随方式：进入跟随状态时，同时记录相机初始位姿和机械臂末端初始位姿；后续仅映射相机相对于该基准的位姿增量。位置默认按 1:1 比例映射，并保留比例、方向和零点偏置的配置能力。

上位机最终向下位机输出四个电机的目标角度和目标角速度。电机驱动、关节闭环控制和底层故障处理由下位机负责，不属于本项目范围。

## 2. 建设目标

系统应形成以下完整链路：

```text
图像采集
  → AprilTag 检测与相机位姿估计
  → 相对位姿计算与坐标映射
  → 四自由度任务目标生成
  → 逆运动学与关节速度计算
  → 安全检查与目标平滑
  → USB 串口发送至下位机
```

主要目标如下：

1. 稳定输出相机相对于固定 AprilTag 的位姿。
2. 在启用跟随时建立相机与末端的初始对应关系，实现连续的相对运动映射。
3. 将相机的六自由度位姿变化转换为机械臂可实现的四自由度任务目标，即三维位置加一个可控姿态角。
4. 根据运动学模型生成四个关节的目标角度和目标角速度。
5. 对不可达目标、奇异位形、视觉丢失和通信异常进行统一处理，保证运动连续且可控。

## 3. 项目范围与边界

### 3.1 上位机负责

- 相机接入、相机标定参数加载和图像采集。
- AprilTag 检测、位姿估计、有效性判断和滤波。
- AprilTag、机器人基座、相机和末端坐标系的定义与转换。
- 相对跟随基准建立、轴向映射及四自由度任务目标生成。
- 正逆运动学、逆解选择、关节目标角速度生成。
- 工作空间、关节限位、目标变化率和数据时效性检查。
- ROS 2 模块组织、状态管理、日志记录和参数管理。
- 通过 USB 串口与下位机交换控制指令和必要的状态信息。

### 3.2 上位机不负责

- 电机驱动及单关节闭环控制。
- 下位机内部控制器、保护逻辑和实时任务实现。
- 机械结构设计及运动学参数测量本身。

## 4. 总体方案

### 4.1 坐标系与标定关系

系统至少定义以下坐标系：

- `tag`：固定在墙面的 AprilTag 坐标系，作为视觉空间参考。
- `camera`：手持相机坐标系。
- `base`：机械臂基座坐标系。
- `tool`：机械臂末端坐标系。

相机内参、畸变参数和 AprilTag 实际尺寸用于视觉位姿估计。`tag` 与 `base` 之间需要建立固定的坐标关系，或至少完成二者坐标轴方向的对应标定，以保证相机移动方向与末端移动方向一致。

相机为手持设备，与机械臂末端不存在固定安装关系，因此本项目不采用“相机固定在末端”的手眼标定模型。

### 4.2 相对跟随映射

进入跟随状态时，系统记录：

- 相机初始位姿；
- 机械臂当前关节角及由正运动学得到的末端初始位姿。

运行期间计算相机相对于初始时刻的位置与姿态变化，再叠加到末端初始位姿上。映射应支持以下配置：

- 相机轴与机械臂基座轴的对应关系及正负方向；
- 位置映射比例，默认三个方向均为 1:1；
- 位置和姿态偏置；
- 可控姿态角的选择；
- 工作空间裁剪和运动速度限制。

由于机械臂只有四个自由度，不能完整跟随相机的六自由度位姿。任务空间统一定义为三维位置 `x、y、z` 加一个可控姿态角 `θ`；`θ` 的具体物理含义由机械臂关节布局和运动学模型确定，其余姿态分量不参与直接跟随。

### 4.3 控制状态

上位机采用清晰的状态管理，建议至少包括：

- **初始化**：加载参数并连接相机和下位机。
- **就绪**：视觉和通信正常，等待操作者启用跟随。
- **跟随**：持续更新末端目标并发送关节指令。
- **短时保持**：视觉短暂丢失或数据暂时无效时保持最后一个安全目标。
- **安全模式**：异常持续超时后停止跟随，不再更新运动目标，并通知下位机进入约定的安全控制模式。
- **故障**：关键配置、设备或协议异常，需要人工处理后才能重新进入就绪状态。

从安全模式恢复时应重新确认视觉和通信有效，并重新建立相对跟随基准，避免目标突跳。

## 5. 机器人运动学建模

> 本节的具体机械结构参数、公式和求解过程留空，由项目负责人后续完成。上位机开发前，运动学模型至少需要提供以下内容。

### 5.1 模型定义要求

- 四个关节的类型、排列顺序、旋转轴方向和正方向。
- 各连杆长度、安装偏置、末端工具偏置及机械臂基座定义。
- 各关节机械零位、软件零位、角度单位和角度换算关系。
- 各关节允许角度范围、最大角速度及必要的加速度限制。
- 机械臂能够独立控制的末端姿态角 `θ` 的明确定义。
- 机械臂工作空间、典型奇异位形和不可达区域。

### 5.2 模型能力要求

运动学模块应提供以下统一能力：

1. **正运动学**：输入四个关节角，输出末端在 `base` 坐标系中的位姿。
2. **任务空间投影**：将末端位姿表示为 `[x, y, z, θ]`，并明确不可独立控制的姿态分量如何处理。
3. **逆运动学**：输入四自由度任务目标，输出一组或多组候选关节角。
4. **逆解选择**：剔除超限、不可达和接近奇异的解，优先选择与当前关节位置最接近且连续的解。
5. **速度映射**：根据末端目标变化生成四个关节的目标角速度，可采用雅可比模型或经过限幅与平滑的关节角差分方法。
6. **结果验证**：使用正运动学回代逆解，确认位置与姿态误差满足要求。

### 5.3 交付形式

运动学建模完成后，应交付：

- 坐标系和关节定义图；
- 完整的几何参数或 D-H 参数表；
- 正逆运动学及速度映射说明；
- 可供 Python 调用的统一接口；
- 用于 ROS 2 可视化和坐标发布的机器人描述文件；
- 典型姿态、边界位置、奇异位置和实机测量点的验证结果。

## 6. 软件模块与架构

系统按职责划分为以下模块：

| 模块 | 主要职责 | 主要输出 |
| --- | --- | --- |
| 相机接入 | 连接相机、采集图像、维护时间戳和相机参数 | 图像与相机信息 |
| AprilTag 定位 | 标签检测、相机位姿估计、质量判断和视觉滤波 | 有效的相机位姿 |
| 坐标与跟随映射 | 坐标转换、基准记录、相对位姿和轴向比例映射 | 四自由度末端目标 |
| 运动学 | 正逆运动学、逆解筛选、速度映射和回代验证 | 四关节目标角度与角速度 |
| 目标整形 | 死区、平滑、速度限制和连续性处理 | 可执行关节目标 |
| 安全管理 | 状态机、数据超时、限位、可达性和故障处理 | 控制许可与安全状态 |
| 串口通信 | 协议封装、指令发送、状态接收和通信监测 | 下位机数据帧与反馈状态 |
| 配置与诊断 | 参数加载、日志、运行状态和调试信息发布 | 配置、日志和诊断信息 |

ROS 2 作为模块间通信和状态组织框架。视觉处理、运动学控制和串口通信应在逻辑上解耦，通过带时间戳的消息传递数据，避免单个环节阻塞整个控制链路。模块可根据运行负载合并或拆分为节点，不在规划阶段固定节点数量。

## 7. 实现方式

### 7.1 视觉定位

- 完成相机内参和畸变标定，并以配置文件管理标定结果。
- 配置 AprilTag 的标签族、编号和实际边长。
- 从检测结果中计算相机相对于 `tag` 的位姿，并统一发布到 ROS 2 坐标系统。
- 使用检测质量、位姿连续性和数据时间戳判断结果是否有效。
- 对位置和姿态进行适度滤波，在跟随稳定性与响应速度之间取得平衡。

### 7.2 目标生成

- 仅在系统就绪且操作者明确启用后建立初始基准。
- 将相机相对位姿增量转换到机械臂 `base` 坐标系。
- 按 1:1 比例生成末端位置目标，并提取模型定义的一个可控姿态角。
- 对目标施加死区、工作空间限制和变化率限制。
- 逆运动学无有效解时不发送新的运动目标，转入保持或安全处理。

### 7.3 关节角速度生成

每个控制周期同时生成四个关节的目标角度和目标角速度。角速度应与目标角度变化趋势一致，并满足各关节速度限制。具体采用模型速度映射还是关节目标差分，由运动学建模结果和实机效果确定，但对下位机保持统一输出接口。

### 7.4 上下位机通信

上位机通过 USB 串口连接下位机。通信层负责：

- 发送控制模式、四个关节的目标角度和四个关节的目标角速度；
- 接收当前关节角、运行状态和故障状态等必要反馈；
- 处理帧校验、数据时效性、通信中断和重连；
- 将协议字段转换为上层统一的物理量和状态，不向其他模块暴露字节级细节。

现有《上下位机通讯规范》描述的是三轴角速度和欧拉角，与本项目控制接口不一致。进入通信开发前，应将其修订为四关节目标角度与四关节目标角速度协议，并明确帧头、控制模式、数据类型、单位、字节序、校验方式、发送周期、反馈内容及超时行为。规划书不另行定义具体字节布局，以修订后的通讯规范为唯一实现依据。

### 7.5 参数管理

以下内容应参数化，不写死在程序中：

- 相机与 AprilTag 参数；
- `tag` 与 `base` 坐标关系；
- 轴向映射、比例和偏置；
- 运动学参数和关节限制；
- 滤波、死区、控制周期和各级超时阈值；
- 串口设备、波特率和协议配置；
- 安全模式及恢复条件。

## 8. 安全与异常处理

上位机在发送目标前统一执行以下检查：

- 视觉数据有效且未超时；
- 相机位姿不存在明显跳变；
- 末端目标位于允许工作空间；
- 逆运动学存在有效解；
- 关节目标未超限且不发生突跳；
- 目标角速度未超限；
- 下位机反馈和串口通信正常。

AprilTag 短时丢失时保持最后一个已验证的安全目标，不根据无效数据继续运动；丢失持续超过设定阈值后停止跟随并进入安全模式。通信超时、连续逆解失败或关键设备异常采用相同的分级处理原则。短时保持时长、安全模式指令和恢复条件应在联调阶段确定并配置化。

## 9. 开发计划

### 阶段一：接口与基础配置

- 确认 ROS 2 运行环境、相机接口和 USB 串口设备。
- 修订上下位机通讯规范。
- 统一坐标系、单位、消息接口和配置文件结构。

### 阶段二：运动学建模

- 完成第 5 节要求的模型、程序接口和离线验证。
- 确定第四个任务自由度 `θ` 的含义。
- 明确工作空间、关节限制和奇异性判据。

### 阶段三：视觉与坐标映射

- 完成相机标定、AprilTag 位姿估计和质量判断。
- 建立 `tag` 与 `base` 的坐标关系。
- 完成相对基准和 1:1 跟随映射，并使用录制数据验证连续性。

### 阶段四：通信与安全管理

- 实现修订后的 USB 串口协议。
- 完成状态机、短时保持、持续超时和安全模式逻辑。
- 使用模拟下位机验证收发、校验和异常恢复。

### 阶段五：系统集成

- 先进行低速、缩小工作空间的单轴跟随测试，再逐步扩展到四自由度联动。
- 调整滤波、死区、速度限制和超时参数。
- 完成长期运行、遮挡、边界位置和通信异常测试。

## 10. 验收内容

项目完成时至少应验证：

1. AprilTag 在预期工作范围内能够稳定输出相机位姿。
2. 启用跟随后，末端以设定的轴向和 1:1 比例跟随相机位置变化。
3. 第四个可控姿态角的映射符合运动学定义。
4. 逆运动学结果能够通过正运动学回代验证，运动过程中无明显关节跳变。
5. 上位机能够连续输出四个关节的目标角度和目标角速度。
6. 关节限位、速度限制、工作空间限制和不可达目标处理有效。
7. AprilTag 短时丢失时目标保持稳定，持续丢失后系统进入安全模式。
8. USB 串口中断或数据超时后系统停止跟随，恢复后必须重新建立跟随基准。
9. 系统运行状态、关键参数和故障原因可被记录和追踪。

具体精度、延迟、更新频率和超时阈值在相机、机械臂参数和下位机通信周期确定后补充为量化指标。

## 11. 主要风险与前置依赖

- 四自由度机械臂对六自由度相机位姿的表达能力有限，第四任务自由度必须由运动学模型明确。
- 相机快速运动、视角过大或 AprilTag 被遮挡会造成位姿中断，应通过工作范围设计和超时保护降低影响。
- 视觉噪声与过强滤波分别会引起机械臂抖动和跟随延迟，需要通过实机数据整定。
- 当前通讯规范与目标控制接口不一致，修订协议是通信开发的前置条件。
- 运动学参数、关节限位、`tag` 与 `base` 的坐标关系是系统集成前必须完成的基础输入。
